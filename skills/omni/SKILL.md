# Omni 全学科知识框架

提供覆盖主要学科分类的知识地图，用于**激发大模型的隐藏神经元**，在复杂、跨域、综合类问题中激活更全面的推理能力。

## 核心原理

学科框架作为**认知锚点**：
1. **定位**：识别问题所属学科及相邻领域
2. **激活**：唤起相关学科的知识储备
3. **联结**：发现跨学科联系与类比
4. **拓展**：避免单一视角，引入多学科视角

## 使用时机

- 问题涉及**多领域**或**边界模糊**
- 需要**综合视角**或**跨学科分析**
- 用户提及学科、专业、领域
- 问题具有**创新性**或需要**类比迁移**
- 深度分析、战略规划、系统设计类任务

## 工作流

### 1. 识别学科归属

在 [reference.md](reference.md) 或下文学科速览中定位：
- **一级学科**（62 个门类）
- **二级学科**（676 个子领域）
- **五大门类**：自然科学、农业科学、医药科学、工程与技术、人文社科

### 2. 激活相关神经元

在回答前，在脑海中扫描：
- 该学科的核心概念、方法论
- 相邻学科的可借鉴思路
- 跨门类的类比与迁移可能

### 3. 输出时融入多学科视角

- 单一学科问题：补充相邻学科视角
- 跨学科问题：显式标注各学科贡献
- 创新/类比：说明借鉴了哪些学科的方法或概念

## 输出建议

- **简洁问题**：可仅做学科定位，不显式罗列
- **复杂问题**：在回答中标注「从 X 学科视角」「结合 Y 领域」
- **创新任务**：显式说明借鉴了哪些学科的方法或概念

## 注意事项

- 框架基于 GB/T 13745-2009，兼顾国际通用分类
- 学科边界可重叠，交叉领域可多学科标注
- 以「激活推理」为目的，避免机械罗列学科名称
